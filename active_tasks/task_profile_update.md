# المهمة: تحديث تصميم ملف المستخدم الشخصي (Profile Page)

## الحالة
- [x] **المرحلة 1: الأساس البصري** (التخطيط، الخلفية، الخطوط) - **مكتملة**
- [x] **المرحلة 2: إعادة هيكلة المكونات** (الترويسة، التبويبات، الإحصائيات) - **مكتملة (تنظيف وأنواع)**
- [ ] **المرحلة 3: عرض المحتوى** (نظام الشبكة، بطاقات الوسائط، الحالات الفارغة) - *قيد التنفيذ (تحسين الشبكة)*
- [ ] **المرحلة 4: التفاعل والإجراءات** (نافذة التعديل، أزرار المتابعة/المراسلة، تحسين محركات البحث SEO)

## الأهداف
تحويل صفحة الملف الشخصي للمستخدم (`/u/[username]`) إلى تجربة بصرية مذهلة وغامرة تتناغم مع جمالية "الخريطة أولاً" (Map-First) الجديدة للوحة التحكم.

## الخطة التفصيلية

### المرحلة 1: الأساس البصري 🎨
- [ ] **خلفية غامرة**: تطبيق `GlobalBackground` (النبض الحي + الجسيمات) على صفحة الملف الشخصي.
- [ ] **تخطيط زجاجي (Glassmorphism)**: استخدام خلفيات شفافة/ضبابية للحاويات للسماح بظهور الخريطة/السمة من خلالها.
- [ ] **حاوية متجاوبة**: ضمان عرض مثالي على الهاتف المحمول (عرض كامل) وسطح المكتب (ممركز بحد أقصى للعرض).

### المرحلة 2: إعادة هيكلة المكونات 🧩
- [ ] **ترويسة الملف (ProfileHeader)**:
    - [ ] صورة رمزية (Avatar) حديثة مع حدود مضيئة (بناءً على الرتبة/الحالة).
    - [ ] خطوط نظيفة للاسم واسم المستخدم.
    - [ ] نظام "شارات" (Badges) للتوثيق، نوع المتجر، والسمعة.
    - [ ] صف إحصائيات مضغوط (المتابعون، المبيعات، والتقييم).
- [ ] **تبويبات الملف (ProfileTabs)**:
    - [ ] تنفيذ رسوم متحركة لتخطيط `framer-motion` عند تبديل التبويبات.
    - [ ] تثبيت الموضع (Sticky) أثناء التمرير.

### المرحلة 3: عرض المحتوى 🖼️
- [ ] **شبكة الوسائط (قصص/منشورات)**: شبكة بأسلوب Instagram مع تأثيرات التمرير (Hover).
- [ ] **بطاقات المنتجات/الطلبات**:
    - [ ] بطاقات مضغوطة تعرض السعر، العنوان، والحالة.
    - [ ] مؤشرات "مباع" / "نشط".
- [ ] **الحالات الفارغة**: رسوم توضيحية/أيقونات إبداعية عندما لا يملك المستخدم محتوى.

### المرحلة 4: التفاعل والإجراءات ⚡
- [ ] **شريط الإجراءات (Action Bar)**:
    - [ ] شريط سفلي عائم للهاتف المحمول (رسالة، اتصال، تحديد موقع).
    - [ ] منطق ظهور زر "تعديل الملف" (للمالك فقط).
- [ ] **البيانات الوصفية و SEO**: وسوم OpenGraph ديناميكية لمشاركة الملفات الشخصية.

## ملاحظات تقنية
- استخدم `framer-motion` لجميع الانتقالات الحركية.
- تأكد من عمل طبقات `z-index` مع شريط التنقل السفلي الثابت.
- تحقق من صحة جلب البيانات (تأكد من معالجة أخطاء 404 بمتانة).
