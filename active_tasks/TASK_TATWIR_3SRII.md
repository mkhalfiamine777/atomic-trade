# TASK_TATWIR_3SRII: خارطة التطوير العصري (Modernization Blueprint)

هذا الملف يحتوي على المهام الاستراتيجية والتطويرية المقترحة للارتقاء ببنية المشروع (Social Commerce) إلى أحدث التقنيات العصرية (State-of-the-Art).

---

## 🗺️ 1. أداء محرك الخرائط (Map Engine Performance)
- [ ] **دراسة الانتقال إلى WebGL:** استبدال `Leaflet` بتقنية تعتمد على معالج الرسومات الجرافيكي (GPU) مثل `Mapbox GL JS` أو هندسة `Deck.gl` (المطورة من Uber).
- [ ] **التنفيذ:** عرض مئات الآلاف من المنتجات (Markers) بسلاسة (60fps) دون استهلاك ذاكرة الهاتف.
- [ ] **تحسينات بصرية (3D):** إضافة بُعد ثلاثي (Pitch & Bearing) متوافق مع تصميم النيون الزجاجي.

## 💾 2. هندسة قواعد البيانات المكانية (Spatial Database Engineering)
- [ ] **تثبيت إضافة PostGIS:** تفعيل الإضافة على خادم `PostgreSQL` الخاص بـ (Railway أو Docker).
- [ ] **تحديث Prisma Schema:** تغيير أنواع الحقول `latitude` و `longitude` من `Float` عادي إلى نوع `Geometry` المكاني الجغرافي.
- [ ] **النتيجة:** تسريع خوارزميات البحث عن المنتجات "المحيطة بالمستخدم" بنسبة 400% بفضل الفهارس المكانية بدلاً من الفحص الخطي.

## 🔍 3. واجهة المستخدم والبحث الذكي (Ultra-modern UI/UX)
- [x] **بناء "Command Menu" زجاجية:** استخدام مكتبة `cmdk` (والمستخدمة من شركة Vercel) لإنشاء لائحة بحث منسدلة فاخرة وسريعة تعمل بالاختصارات (مثل `Ctrl+K`).
- [x] **ربط البيانات:** جلب تصنيفات المنتجات (إلكترونيات، طعام، إلخ) من ملف `JSON` محلي للسرعة، أو ديناميكياً من `Prisma`.
- [ ] **تحسين الحالة (State Management):** استبدال Context API الثقيل المحيط بالخريطة بمكتبة خفيفة مثل `Zustand` لضمان استجابة لحظية.

## 🤖 4. الذكاء الاصطناعي في المبيعات (AI-Driven Commerce)
- [ ] **البحث الدلالي (Semantic Search):** استخدام الذكاء الاصطناعي (Vector Embeddings) للبحث.
- [ ] **التنفيذ:** السماح للمستخدم بالبحث المتقدم (مثال: "أريد هدية لعيد ميلاد طفل") بدلاً من البحث بالكلمات المطابقة فقط.
- [ ] **قاعدة البيانات:** دمج إضافة `pgvector` مع الواجهة الخلفية لـ Prisma.

## 🛡️ 5. اقتصاد الثقة والمناطق (Gamified Trust & Zone Master)
- [ ] **تطوير ZoneMaster:** ربط نظام الحكام للمناطق الافتراضية بنظام تقدم (Leveling System) أكثر عمقاً.
- [ ] **ترميز الثقة (Tokenization):** (اختياري مستقبلاً) استخدام مفاهيم Web3 المبسطة لمنح المستخدمين إحساساً بملكية شارات الثقة (Trust Badges) أو العقارات الافتراضية داخل اللعبة.

## 🔗 6. نظام المطابقة الذكية بين البائع والمشتري (Smart Product Matching)
- [x] **كتالوج المنتجات الهرمي:** إنشاء ملف `products_catalog.json` يحتوي على فئات رئيسية وتصنيفات فرعية.
- [x] **مكون القوائم المتتالية (Cascading Dropdowns):** بناء مكون `CascadingProductSelect.tsx` يعرض قائمة الفئة والتصنيف.
- [x] **تحديث نافذة "إضافة منتج":** استبدال حقل النص الحر بالمكون الجديد.
- [x] **تحديث نافذة "اطلب ما تحتاج":** نفس القوائم المتتالية الموحدة.
- [x] **محرك المطابقة (Matching Engine):** عند إنشاء عرض/طلب، يبحث الخادم عن مطابقات (نفس الفئة + التصنيف).
- [x] **تحديث قاعدة البيانات:** إضافة حقل `subcategory` لجدول `Listing` في Prisma.

---

## 🛠️ 7. لوحة تحكم إدارية (Admin Panel)
- [ ] **نقل الكتالوج إلى قاعدة البيانات:** إنشاء جدولي `Category` و `Subcategory` في Prisma بدلاً من ملف JSON.
- [ ] **صفحة إدارية محمية `/admin`:** واجهة لإضافة/تعديل/حذف الفئات والتصنيفات من المتصفح مباشرة.
- [ ] **صلاحيات المشرف:** حماية المسار بدور `ADMIN` في نظام المصادقة.
