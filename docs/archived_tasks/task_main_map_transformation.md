# خطة تحويل لوحة التحكم إلى خريطة تفاعلية (Dashboard Map Transformation)

## 1. الهدف (Objective)
تحويل صفحة لوحة التحكم (`/dashboard`) لتصبح **الواجهة الرئيسية للتطبيق (Map-First Dashboard)**.
الصفحة الرئيسية (`/`) ستبقى كصفحة هبوط تسويقية، بينما `/dashboard` ستكون هي "التطبيق" الفعلي الذي يرتكز على الخريطة.

---

## 2. تحليل المكونات الحالية (Current Component Analysis)
سنستخدم المكون القوي `src/components/Map.tsx`، وسنقوم بتعديله أو تغليفه ليعمل بملء الشاشة داخل لوحة التحكم.

### التعديلات المطلوبة:
1.  **إزالة هوامش لوحة التحكم:** التأكد من أن `DashboardLayout` (إن وجد) يسمح بوضع ملء الشاشة، أو إنشاء Layout خاص للخريطة.
2.  **تكامل الخريطة:**
    - جعل الخريطة تملأ حاوية الصفحة (`h-[calc(100vh-64px)]` أو `h-screen`).
    - ضبط `z-index` لتكون العناصر التفاعلية فوق الخريطة.

---

## 3. خطة التنفيذ (Implementation Steps)

### المرحلة 1: إعداد صفحة التوجيه (`src/app/dashboard/page.tsx`)
1.  تنظيف محتوى الصفحة الحالي.
2.  استدعاء مكون `Map` (محاط بـ `ClientOnly` Wrapper لتجنب مشاكل SSR).
3.  إضافة `BottomNav` للتنقل السلس (يظهر فقط في الموبايل).
4.  هيكلة الصفحة:
    ```tsx
    <div className="relative h-screen w-full overflow-hidden">
        <Map currentUserId={user.id} />
        <BottomNav />
    </div>
    ```

### المرحلة 2: تحسين تجربة الملاحة
1.  **زر العودة:** إضافة زر عائم للعودة للقائمة الرئيسية أو الملف الشخصي.
2.  **شريط الأدوات العائم:** دمج أدوات البحث والفلتر (`MapFilterBar`) بشكل عائم فوق الخريطة.

### المرحلة 3: ربط البيانات
1.  جلب بيانات المستخدم الحالي (`currentUserId`) وتمريرها للخريطة.
2.  التأكد من عرض الإشعارات والقصص فوق طبقة الخريطة.

---

## 4. سيناريوهات المستخدم (User Scenarios)

### سيناريو الدخول (للأعضاء):
- المستخدم يضغط "دخول للأعضاء" في الصفحة الرئيسية.
- يتم توجيهه إلى `/dashboard`.
- تظهر له الخريطة التفاعلية فوراً، مركزة على موقعه، مع عرض المتاجر والمنشورات القريبة.

### سيناريو الزائر (Guest / استكشاف):
- المستخدم يضغط "إبحث في الخريطة" في الصفحة الرئيسية.
- يتم توجيهه إلى `/dashboard` (أو `/map`).
- تظهر الخريطة **بوضع القراءة فقط**:
    - يمكنه التصفح، التكبير، ورؤية تفاصيل المنتجات.
    - عند محاولة التفاعل (مراسلة، إعجاب، نشر) -> تظهر نافذة تطلب تسجيل الدخول.


---

## 5. المخاطر والحلول (Risks & Mitigations)
- **تعارض الـ Layout:** إذا كان هناك `Sidebar` ثابت في لوحة التحكم، قد يحتاج لإخفاء عند عرض الخريطة على الجوال.
- **الأداء:** تحميل الخريطة قد يكون ثقيلاً؛ سنستخدم `Loading Skeleton` أثناء جلب البيانات.
