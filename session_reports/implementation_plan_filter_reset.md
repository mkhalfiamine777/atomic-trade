# 📋 خطة تنفيذ: منطق "إعادة التعيين التلقائي" (Auto-Reset Logic)

## 🎯 الهدف
تعديل سلوك أزرار التصفية بحيث:
عندما يقوم المستخدم باختيار **جميع** الفلاتر الأربعة (فيديو، صور، منتجات، طلبات)، يعود النظام تلقائياً إلى الحالة الأصلية (عرض الكل واختفاء تظليل الأزرار).

## 📍 أماكن التعديل (Modification Points)

### 1. ملف `src/components/MapFilterBar.tsx`
*   **الإجراء:** تعريف وتصدير ثابت `ALL_FILTERS`.
*   **الهدف:** ضمان وجود "مصدر حقيقة واحد" (Single Source of Truth) لقائمة الفلاتر، لاستخدامه في المقارنة.
*   **الكود المقترح:** 
    ```typescript
    export const ALL_FILTERS: FilterType[] = ['VIDEO', 'IMAGE', 'PRODUCT', 'REQUEST']
    ```

### 2. ملف `src/components/Map.tsx`
*   **الإجراء:** تعديل دالة `toggleFilter`.
*   **المنطق الجديد:**
    1.  حساب الحالة القادمة للفلاتر (`nextFilters`).
    2.  مقارنة طول `nextFilters` مع طول `ALL_FILTERS`.
    3.  إذا كانا متساويين (المستخدم اختار الكل) -> نقوم بتشغيل "إعادة التعيين" وتفريغ القائمة `[]`.
    4.  خلاف ذلك -> نطبق التغيير الطبيعي.
*   **التأثير على الأيقونات:** بما أن منطق عرض الخريطة يعتمد على `selectedFilters`، فإن العودة إلى `[]` ستجعل الخريطة تظهر **جميع الأيقونات** تلقائياً (كما كانت في الأصل).

## ✅ التحقق (Verification)
1.  تحديد 3 فلاتر -> الخريطة تظهر 3 أنواع فقط.
2.  تحديد الفلتر الرابع -> الخريطة "تومض" وتعود لعرض كل شيء، وتختفي الألوان من جميع الأزرار.
