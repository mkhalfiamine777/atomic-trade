# 👑 خطة تنفيذ: سيطرة المناطق (Zone Master) - "لعبة العروش"

## 🎯 الهدف
تحويل الخريطة إلى رقعة شطرنج تنافسية. كل منطقة (Zone) لها "سيد" (Lord) يجمع الضرائب (افتراضياً) أو يحصل على الظهور المميز.

## 🛠️ الآلية التقنية (Technical Approach)

### 1. تقسيم الخريطة (The Grid)
سنستخدم **GeoHash** بتقريب 6 (Precision 6).
*   **الحجم التقريبي:** ~1.2 كم × 0.6 كم للمربع الواحد.
*   **الميزة:** تقسم العالم تلقائياً لشبكة ثابتة دون الحاجة لرسم حدود يدوية.

### 2. نموذج البيانات (Database Schema)
نحتاج لتحديث `schema.prisma` لإضافة نظام تتبع النقاط:

#### أ. تحديث `ZoneMaster` (المناطق المحتلة)
نضيف حقل `currentScore` لمعرفة النقاط المطلوب كسرها لاحتلال المنطقة.

#### ب. إنشاء `ZoneActivity` (المنافسون)
جدول جديد يتتبع نشاط كل مستخدم في كل منطقة.
```prisma
model ZoneActivity {
  id        String   @id @default(uuid())
  geoHash   String
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  score     Int      @default(0) // نقاط السيطرة
  lastActive DateTime @default(now())

  @@unique([geoHash, userId]) // سجل واحد لكل مستخدم في المنطقة
  @@index([geoHash, score])   // لسرعة الاستعلام عن المتصدر
}
```

### 3. قوانين اللعبة (The Rules) 📜
كيف تجمع النقاط تحتل المنطقة؟

*   **إنشاء متجر/إعلان:** +50 نقطة.
*   **الحصول على إعجاب:** +10 نقاط.
*   **بيع منتج (مستقبلاً):** +100 نقطة.

**آلية الانقلاب (The Coup):**
عندما يتجاوز مستخدم نقاط "السيد الحالي" في `ZoneActivity`:
1.  يتم تحديث `ZoneMaster` بالسيد الجديد.
2.  يتم إرسال إشعار للملك المخلوع: "🚨 لقد فقدت السيطرة على منطقتك!"
3.  يتم إرسال إشعار للملك الجديد: "👑 أنت الآن سيد حي [اسم الحي]!"

### 4. واجهة المستخدم (The UI)
*   **طبقة المناطق (Zone Layer):** رسم مستطيلات شفافة على الخريطة تمثل المناطق (اختياري، قد يكون مزدحماً).
*   **أيقونة الملك:** إظهار صورة "السيد" وعليها تاج 👑 في مركز المنطقة الجغرافية (Center of GeoHash).
*   **بطاقة المنطقة:** عند الضغط على أي مكان فارغ في الخريطة، تظهر نافذة:
    > "أنت في منطقة [اسم المنطقة]
    > المسيطر: [صورة واسم الملك]
    > نقاط السيطرة: 450"

## 📝 خطوات التغيير
1.  تحديث `schema.prisma` وتشغيل `prisma migrate`.
2.  إنشاء `ZoneService` لحساب الـ GeoHash وتحديث النقاط.
3.  تطوير خطاف `useZoneMaster` لجلب بيانات المناطق وعرضها على `Map.tsx`.
