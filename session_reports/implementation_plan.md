# خطة التنفيذ - منصة التجارة الاجتماعية "الربح أولاً"

## وصف الهدف
بناء منصة تجارة اجتماعية ضخمة الحجم حيث تم تصميم كل تفاعل لتسهيل التجارة وتوليد الأرباح للمستخدمين. يجب أن يدعم النظام التزامن العالي (للميزات الاجتماعية) والنزاهة المالية (للميزات التجارية).

## مطلوب مراجعة المستخدم
> [!IMPORTANT]
> **تحدي الـ 300 متر (The 300m Challenge):** ميزة "تنبيه القرب" تتطلب تتبعاً مستمراً للموقع (Background Geolocation).
> **الحل التقني:** سنبدأ بتطبيق الويب (PWA) للميزات الأساسية، ولكن **التطبيق الأصلي (Native App)** هو شرط أساسي لعمل التنبيهات في الخلفية بدقة.
> **آلية التنبيه:** (Server-side Geofencing) -> الخادم يحسب المسافة ويرسل Push Notification عند تحقق شرط < 300m.

> **آلية التنبيه:** (Server-side Geofencing) -> الخادم يحسب المسافة ويرسل Push Notification عند تحقق شرط < 300m.

> [!TIP]
> **الميزات العبقرية 2.0 (Viral Mechanics):** تم اعتماد حزمة "النمو الفيروسي" التالية:
> 1.  **مول الحومة (Digital Fiefdom):** نظام ولاء جغرافي (عمولة لمن يسيطر على الحي).
> 2.  **البائع المتجول الذكي (Live Tracking):** أيقونة متحركة للباعة المتجولين (يتطلب تحديث موقع كل 10 ثوانٍ).
> 3.  **المزاد العكسي (Reverse Wishlist):** المشتري يطلب، والبائعون يتنافسون.
> 4.  **صناديق الكنز (Geo-Looting):** ميزة AR للبحث عن جوائز في الشارع.
> 5.  **الصدقة الجغرافية (Geo-Charity):** دفع ثمن منتج ليأخذه المحتاجون.

## المكدس التقني المقترح (Tech Stack) - محدث للطبيعة الجغرافية
- **الواجهة الأمامية (Web):** Next.js 14 - لإدارة الداشبورد والعرض العام.
- **تطبيق الجوال (Mobile):** مطلوب مستقبلاً (Flutter أو React Native) <- **ضروري جداً لميزة تنبيه 50 متر**.
- **الخرائط (Maps SDK):** Google Maps Platform أو Mapbox (الأفضل للتخصيص).
- **قاعدة البيانات:** PostgreSQL + **PostGIS Extension** (لمعالجة الاستعلامات الجغرافية المعقدة).
- **الوقت الحقيقي (Real-time):** Socket.io أو Pusher (لتتبع حركة المستخدمين وإرسال التنبيهات الفورية).

## التغييرات المقترحة (المرحلة 1: التأسيس الجغرافي)

### هيكلية المشروع
سنعتمد هيكلية مبنية على **الموقع (Location-First)**.
`src/features/map`, `src/features/tracking`, `src/features/marketplace`.

#### [جديد] [package.json](file:///e:/Social_Commerce_Project/package.json)
- تهيئة تطبيق Next.js باستخدام `create-next-app`.

#### [جديد] [schema.prisma](file:///e:/Social_Commerce_Project/prisma/schema.prisma)
- نمذجة `UserType` (INDIVIDUAL, SHOP, COMPANY).
- إضافة حقل `location` (خط الطول/العرض) بنوع بيانات PostGIS.
- **نموذج الربح (وتخفيف مخاطر البيع المباشر):**
    - **الأساس:** الاشتراك الشهري `SubscriptionTier` (للمحلات والشركات) للظهور على الخريطة.
    - **البيع المباشر (Cash):** لا يمكننا فرض عمولة عليه. البديل: نموذج "الدفع مقابل التواصل" (Pay-per-Lead) أو "نقاط كشف الرقم".
    - **البيع الرقمي:** عمولة `CommissionRate` تلقائية عند الدفع عبر التطبيق (مقابل ضمان حق المشتري).
    - **التحفيز:** نقاط ولاء (Loyalty Points) للمشتري *فقط* إذا قام بمسح QR Code البائع (إثبات البيع).
- **نموذج الميزات العبقرية:**
    - حقل `isMobileVendor` للمستخدمين المتجولين.
    - جدول `ZoneMaster` لتتبع "مول الحومة".
    - جدول `CharityItem` للمنتجات المدفوعة مسبقاً.
