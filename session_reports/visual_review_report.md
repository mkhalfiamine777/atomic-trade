# 👁️ تقرير المراجعة المرئية والبرمجية (Visual Audit Report)

## 🩺 التشخيص (Diagnosis)
بناءً على طلبك، قمت بإجراء فحص دقيق "تخيلي" للكود (Code walkthrough) لتوقع الأخطاء المرئية التي قد تحدث أثناء الاستخدام الفعلي.

### 🟥 الخطأ المكتشف (Critical Visual Bug)
**الوصف:** "تسرب حالة التمدد" (Spiderfy State Leakage).
**المكان:** ملف `Map.tsx` - السطر 308.
**السبب:** استخدام `key={i}` (فهرس المصفوفة) عند رسم المجموعات (`SmartMarkerGroup`).
**السيناريو:**
1.  المستخدم يفتح "مجموعة A" (فتتمدد الأيقونات).
2.  يقوم المستخدم بتغيير التصفية (Filter) لإزالة بعض العناصر.
3.  يختفي "مجموعة A"، ويحل محله "مجموعة B" في نفس الفهرس `index`.
4.  **النتيجة المرئية:** "مجموعة B" تظهر **مفتوحة تلقائياً** (وهو خطأ)، أو يحدث "وميض" مزعج، لأن React يظن أنه نفس المكون.

### 🟩 التصحيح (The Fix)
*   **الإجراء:** استبدال `key={i}` بـ `key={coordinates}`.
*   **النتيجة:** الآن، كل مجموعة مرتبطة بموقعها الجغرافي حصراً. إذا تغيرت التصفية، النظام يفهم أن هذه "مجموعة جديدة" ويعيد رسمها وهي "مغلقة" (Collapsed). هذا يضمن استقرار الواجهة.
