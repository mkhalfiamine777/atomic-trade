# 🏗️ دليل المعمارية — منصة التجارة الاجتماعية

> **للمطورين الجدد الذين ينضمون للمشروع.** هذا المستند يشرح كيفية تنظيم النظام.

## نظرة عامة

التجارة الاجتماعية هي **سوق جغرافي اجتماعي** حيث يتفاعل المستخدمون والمتاجر والشركات على خريطة حية. كل نشاط مرتبط بإحداثيات جغرافية.

## المكدس التقني

| الطبقة | التقنية | الغرض |
|--------|---------|-------|
| إطار العمل | Next.js 16 (App Router) | عرض الخادم، التوجيه، عمليات الخادم |
| الواجهة | React 19 + TailwindCSS | المكونات، التنسيق |
| الخريطة | Leaflet + Supercluster | التصور الجغرافي، التجميع |
| قاعدة البيانات | PostgreSQL + Prisma | تخزين البيانات، ORM |
| الوقت الفعلي | Socket.io | الدردشة، الإشعارات، التحديثات الحية |
| رفع الملفات | UploadThing | رفع الصور والفيديوهات |
| المصادقة | كوكيز (`user_id`) | إدارة الجلسات البسيطة |

## القواعد المعمارية الأساسية

### 1. قاعدة الموقع الأساسية (Cardinal Location Rule)
كل أيقونة مستخدم مرتبطة بـ **آخر إحداثيات GPS معروفة له**. جميع الأنشطة تُختم بموقع المستخدم اللحظي.

- **الأفراد (INDIVIDUAL):** يتم تحديث الموقع كل 20 ثانية عبر استطلاع GPS.
- **المتاجر والشركات (SHOP/COMPANY):** الموقع **ثابت** (قانون التثبيت). الأنشطة تدور على بعد 10-20 متراً حول المتجر (الانحراف المداري).

### 2. عمليات الخادم فقط (Server Actions Only)
جميع عمليات قاعدة البيانات تمر عبر عمليات الخادم في `src/actions/`. **لا تستدعِ** Prisma أبداً من مكونات العميل.

```typescript
// ✅ صحيح
'use server'
export async function createListing(...) {
    const [result] = await db.$transaction([...])
    return { success: true, data: result }
}

// ❌ خطأ — لا ترمِ أخطاءً، ولا تستخدم Prisma في العميل
throw new Error('...')
```

### 3. نمط معالجة الأخطاء
جميع عمليات الخادم تُرجع `{ success: boolean, error?: string }` — لا تستخدم `throw` أبداً.

### 4. أمان الأنواع (Type Safety)
- ممنوع `as any` — ممنوع `error: any`
- استخدم `error: unknown` مع `instanceof Error`
- جميع الأنواع تعيش في `src/types/index.ts`

### 5. لغة التصميم
- السمة الداكنة: `bg-zinc-950` كخلفية أساسية
- ألوان النيون المُبرزة: ذهبي (متاجر)، سماوي (أفراد)، بنفسجي (شركات)
- تأثيرات زجاجية (Glassmorphism) + حركات دقيقة

## النشر والتوزيع

- **المنصة:** Railway (نشر تلقائي عند الدفع إلى فرع `main`)
- **CI/CD:** GitHub Actions يشغّل فحص الأنواع → الاختبارات → البناء عند كل دفع
- **قاعدة البيانات:** PostgreSQL على Railway (المخطط يتزامن عبر `prisma db push` عند بدء التشغيل)

## القرارات التصميمية الرئيسية

| القرار | المبرر |
|--------|--------|
| خادم موحد في `server.ts` | تبسيط النشر إلى خدمة واحدة على Railway |
| مصادقة بالكوكيز (بدون NextAuth) | أبسط للنسخة الأولية؛ مصادقة بالهاتف |
| `$transaction` لجميع عمليات الكتابة | منع تعارضات السباق في العمليات المتزامنة |
| Supercluster للخريطة | يتعامل مع آلاف العلامات بكفاءة |
| استطلاع GPS كل 20 ثانية | توازن بين الدقة واستهلاك البطارية/النطاق الترددي |
| قانون التثبيت (Anchor Rule) | منع المتاجر من الانجراف مع حركة هاتف المالك |
| الانحراف المداري (Orbital Spread) | منع تكدس أيقونات المنتجات فوق أيقونة المتجر |
